◆ pin_expiry

string Nexmo.Api.NumberVerify.VerifyRequest.pin_expiry
getset

The PIN validity time from generation. This is an integer value between 60 and 3600 seconds. The default is 300 seconds. When specified together, pin_expiry must be an integer multiple of next_event_wait. Otherwise, pin_expiry is set to equal next_event_wait. For example: pin_expiry = 360 seconds, so next_event_wait = 120 seconds - all three attempts have the same PIN. pin_expiry = 240 seconds, so next_event_wait = 120 seconds - 1st and 2nd attempts have the same PIN, third attempt has a different PIN. pin_expiry = 120 (or 200 or 400 seconds) - each attempt has a different PIN.
